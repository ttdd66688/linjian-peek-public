# 掌心窗 v0.1.9 · 小红书冷启动与 UI 优化

## 修复点

### 1. open_app 冷启动增强

此前小红书在后台时可以唤醒，但清后台后可能无法冷启动。v0.1.9 在 Android 端增强启动链路：

1. `PackageManager.getLaunchIntentForPackage(pkg)` 标准启动；
2. 失败时查询 `ACTION_MAIN + CATEGORY_LAUNCHER` 的真实入口 Activity；
3. 使用显式 Activity 再次启动；
4. 回传更明确的结果：`opened_standard` / `opened_launcher_activity` / `no_launch_intent` / `activity_start_failed` / `package_not_found`。

这样即使仍被系统或厂商后台限制拦截，也能知道失败原因，而不是只看到 pending。

### 2. MCP open_app 等待回传

MCP 的 `open_app` 工具现在会在排队后等待短时间回查 `/api/command/status`，如果手机端及时回传，就直接返回 completed/failed；如果未回传，会明确返回 queued/pending，而不是让人误以为没有发出去。

### 3. UI 优化

- 底部导航高度从 74dp 降到 62dp；
- 多数按钮高度从 50-56dp 降到 44-46dp；
- 卡片圆角和内边距减小；
- 标题字号略收紧；
- 调试状态文案保持，但整体页面更轻。
